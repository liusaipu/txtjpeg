# TxtJPEG 设计文档

## 1. 系统架构

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   input.txt     │────▶│  Parser/Scanner  │────▶│  float32 array  │
└─────────────────┘     └──────────────────┘     └────────┬────────┘
                                                          │
                    ┌─────────────────────────────────────┘
                    ▼
┌────────────────────────────────────────────────────────────────────┐
│                         Compress Pipeline                           │
│  1. infer format token (%9.2f,)                                    │
│  2. streaming parse → array('f')                                   │
│  3. compute vmin/vmax                                              │
│  4. linear normalize to uint8: (v-vmin)/(vmax-vmin)*255            │
│  5. reshape to 2D (auto or --shape), pad to 8× multiple            │
│  6. Pillow grayscale JPEG encode (quality, subsampling)            │
│  7. pack into .tj container                                        │
└────────────────────────────────────────────────────────────────────┘
                                                          │
                                                          ▼
                                                  ┌───────────────┐
                                                  │   output.tj   │
                                                  └───────┬───────┘
                                                          │
┌────────────────────────────────────────────────────────────────────┐
│                        Decompress Pipeline                          │
│  1. read .tj header (magic/version/quality/shape/N/vmin/vmax/columns/fmt) │
│  2. decode JPEG to grayscale numpy array                                    │
│  3. inverse normalize: pixel/255*(vmax-vmin)+vmin                  │
│  4. format each value with saved fmt string                        │
│  5. restore line breaks according to `columns`                     │
│  6. streaming write to output.txt                                  │
└────────────────────────────────────────────────────────────────────┘
```

## 2. 模块设计

### 2.1 `txtjpeg.py`

单一入口文件，分为以下逻辑区域：

| 函数/类 | 职责 |
|---------|------|
| `_infer_format(path)` | 从首个 token 推断原始浮点格式（如 `%9.2f,`） |
| `_iter_floats(path)` | 流式解析文本中的浮点数，内存友好 |
| `_compute_shape(n, shape)` | 自动或显式确定二维图像尺寸，8 的倍数对齐 |
| `compress(...)` | 完整压缩流程，返回统计信息 |
| `decompress(...)` | 完整解压流程，返回统计信息 |
| `main()` | argparse CLI 入口 |

### 2.2 解析器

- 以 1 MiB 为单位分块读取文件。
- 维护 `carry` 缓冲区，处理跨块边界 token。
- 按逗号切分，使用 `float(tok.strip())` 转数值。
- 避免一次性加载整个 579 MB 文件或创建 5800 万个 Python 字符串对象。

### 2.3 归一化与反映射

- 线性映射：
  ```
  pixel = round((value - vmin) / (vmax - vmin) * 255)
  ```
- 反映射：
  ```
  value = pixel / 255 * (vmax - vmin) + vmin
  ```
- `vmin == vmax` 时全部映射为 0，解压后统一返回 `vmin`。

### 2.4 二维化策略

默认策略：

```
w = ceil(sqrt(N))，向上对齐到 8 的倍数
h = ceil(N / w)，向上对齐到 8 的倍数
```

- 优先保证接近正方形，以提升 JPEG 压缩效率。
- 末尾不足像素用 `vmin` 对应的灰度值 0 填充。
- 可选 `--shape HxW` 显式指定，语义为 **行数 × 列数**（rows × cols）。例如 `--shape 10000x5789` 表示 10000 行、5789 列。

### 2.5 JPEG 编码

- 使用 Pillow `Image.fromarray(arr, mode='L')` 生成灰度图。
- 参数：
  - `quality`：1–100，默认 85。
  - `subsampling='4:4:4'`：灰度图影响有限，但保留最高质量。
  - `optimize=True`：启用 JPEG 哈夫曼表优化。

## 3. 容器格式（.tj）

小端序二进制结构：

| 字段 | 类型 | 长度 | 说明 |
|------|------|------|------|
| Magic | bytes | 8 | `TXTJPEG\0` |
| Version | uint8 | 1 | 当前为 2 |
| Quality | uint8 | 1 | JPEG quality（1–100） |
| Width | uint32 | 4 | 图像宽度 |
| Height | uint32 | 4 | 图像高度 |
| N | uint64 | 8 | 原始浮点个数 |
| vmin | float64 | 8 | 最小值 |
| vmax | float64 | 8 | 最大值 |
| columns | uint32 | 4 | 每行 token 数，0 表示无换行 |
| fmt_len | uint16 | 2 | 格式字符串长度 |
| fmt_str | bytes | fmt_len | 例如 `%9.2f,` |
| jpeg_len | uint64 | 8 | JPEG 数据长度 |
| jpeg_data | bytes | jpeg_len | 标准 JPEG 字节流 |

### 3.1 设计理由

- 自包含：无需外部元数据即可解压。
- 标准 JPEG：容器内的 `jpeg_data` 可直接存为 `.jpg` 查看。
- 紧凑：元数据仅几十字节。

## 4. CLI 设计

```bash
python txtjpeg.py compress  <input.txt> <output.tj> [--quality N] [--shape HxW]
python txtjpeg.py decompress <input.tj> <output.txt>
```

### 4.1 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `quality` | int | 85 | JPEG 质量 1–100 |
| `--shape` | HxW | 自动 | 显式指定二维形状 |
| `--subsampling` | str | 4:4:4 | JPEG 下采样模式 |

## 5. 误差来源分析

1. **uint8 量化误差**：最大约 `(vmax - vmin) / 255`。
2. **JPEG DCT 量化误差**：随 quality 降低而增大，会引入块效应和局部平滑。
3. **格式化误差**：解压后按原模板四舍五入输出，属于二次量化。

对当前样例文件，quality=85 时：

- 最大绝对误差 ≈ 213（相对 5.1%）
- 平均绝对误差 ≈ 14.5

## 6. 性能与资源

在 `result_all_weight_d_正式.txt`（579 MB）上：

| 阶段 | 耗时 | 峰值内存估算 |
|------|------|--------------|
| 解析 + 归一化 | ~4.8 s | 输入文件（579 MB）+ float32 数组（232 MB）+ uint8 图像（58 MB） |
| JPEG 编码 | <1 s | Pillow 内部缓冲 |
| 解压 | ~6.3 s | 图像（58 MB）+ float64 数组（464 MB）+ 输出写入 |

整体可在普通笔记本上秒级完成压缩/解压。

## 8. 局限与注意事项

- 解压后的数值**不保证精确还原**。
- 当前仅支持固定宽度浮点格式；动态宽度/科学计数法需扩展 `_infer_format`。
- 数据形状未知时采用自动推断，可能会破坏原始矩阵的某些局部相关性。
- 换行信息通过 `columns` 字段保存；若文件换行不规则，只能保留第一行的列数。
